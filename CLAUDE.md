# GoalFlow — Система управления задачами и временем

## О проекте

Учебный проект (Модуль 3 — Персонализация). Минималистичный таск-менеджер с иерархией Цели → Проекты → Задачи, встроенным учётом времени и локальным хранением данных. Выбор стека, архитектуры и способа реализации — полностью свободный.

Полное ТЗ: `GoalFlow_project.md`

## Контекст проекта

- `docs/00-brief.md` — цель, аудитория, ограничения, критерии качества
- `docs/01-rules.md` — правила взаимодействия с ассистентом
- `docs/02-knowledge/glossary.md` — термины и сущности проекта
- `docs/03-templates.md` — промпты для типовых задач разработки
- `docs/04-decisions-log.md` — лог принятых решений

## Правила работы

- Перед предложением решения — уточнить контекст, если он неоднозначен
- Не выдумывать факты о библиотеках и API: если не уверен — сказать прямо
- Предлагать минимально достаточное решение, не усложнять без необходимости
- Если я ошибаюсь — говорить прямо, с примером и альтернативой
- Сначала архитектура/подход — потом код. Не писать код без согласования подхода на нетривиальных задачах
- Всегда фиксировать важные решения в `docs/04-decisions-log.md`

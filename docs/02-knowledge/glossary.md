# 02-knowledge: Глоссарий GoalFlow

## Сущности предметной области

**Goal (Цель)**
Верхний уровень иерархии. Стратегический результат, к которому стремится пользователь. Пример: «Запустить продукт до марта». Имеет прогресс, вычисляемый из вложенных задач.

**Project (Проект)**
Средний уровень. Группа связанных задач, объединённых общим результатом. Принадлежит одной цели (или существует без привязки). Пример: «MVP лендинга».

**Task (Задача)**
Базовая единица работы. Имеет статус, опциональные дедлайн и теги, встроенный учёт времени. Принадлежит проекту (или существует без привязки). Пример: «Написать текст для hero-секции».

**Subtask (Подзадача)**
Опциональный четвёртый уровень. Декомпозиция задачи. Обсуждается: нужен ли отдельный уровень или достаточно чеклиста внутри задачи.

**Time Entry (Запись времени)**
Единица учёта времени: старт → стоп (через таймер) или ручной ввод продолжительности. Привязана к конкретной задаче.

## Статусы задачи

- `todo` — запланирована
- `in_progress` — в работе (таймер может быть активен)
- `done` — завершена
- `cancelled` — отменена (не учитывается в прогрессе)

## UI-концепции

**Quick Capture**
Быстрый ввод задачи без открытия полной формы. Вызывается хоткеем. Создаёт задачу с минимумом полей (название + inbox).

**Command Palette**
Глобальный поиск и быстрые действия через Cmd+K. Аналог: VS Code, Linear.

**Inbox**
Особый «проект» для задач без привязки к цели/проекту. Точка сбора задач перед организацией.

**View (Вид)**
Способ отображения задач: список (`list`), канбан (`kanban`), возможно календарь (`calendar`). Может быть сохранён с набором фильтров.

**Filter (Фильтр)**
Условие отбора задач: по статусу, проекту, цели, тегу, дедлайну, времени. Фильтры комбинируются.

## Технические термины

**Local-first**
Подход: данные хранятся локально (в браузере), приложение работает без интернета. Облако опционально.

**IndexedDB**
Встроенная в браузер база данных для хранения структурированных данных. Альтернатива localStorage для больших объёмов.

**SQLite WASM**
SQLite, скомпилированный в WebAssembly. Позволяет использовать SQL в браузере без бэкенда.

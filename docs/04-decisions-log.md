## 2026-05-01: Создание проектного контура

**Решение:** создана структура контекста по модели из урока 5.1 — CLAUDE.md + docs/ с 5 файлами
**Почему:** проект будет развиваться больше недели, нужен устойчивый контекст для Claude Code; без него каждый новый чат требует повторного объяснения правил и терминов
**Отвергнуто:** держать контекст только в голове / в одном большом промпте — не масштабируется

## 2026-05-01: Модель данных — иерархия сущностей

**Решение:** Goal → Project → Task → Subtask (4 уровня). Subtask реализован через `parentId?` в Task, а не отдельной сущностью. Теги — отдельная сущность Tag + связь TaskTag.
**Почему:** parentId элегантнее — не дублирует логику, глубина ограничена одним уровнем программно. Отдельная сущность Tag даёт color и централизованное управление.
**Отвергнуто:** Subtask как отдельная таблица (дублирование), чеклист внутри Task (теряем статусы и время подзадач)

## 2026-05-01: Модель данных — связи между сущностями

**Решение:** Task принадлежит одному Project (one-to-many). Project принадлежит одной Goal (one-to-many). Task без Project попадает в виртуальный Inbox.
**Почему:** many-to-many порождает проблемы UI (где показывать задачу?), прогресса (дублирование) и учёта времени. Linear, Things 3 — все используют one-to-many, и этого достаточно.
**Отвергнуто:** many-to-many Task ↔ Project — overengineering для данного проекта

## 2026-05-01: Стек — фреймворк и хранилище

**Решение:** React + Vite + TypeScript + Dexie.js (IndexedDB)
**Почему:** React знаком разработчику; Vite — быстрый старт и HMR; Dexie — удобная обёртка над IndexedDB с поддержкой реактивных запросов (useLiveQuery); TypeScript даёт типобезопасность модели данных.
**Отвергнуто:** Vue 3 (незнакомо), SQLite WASM (сложная настройка WASM/заголовков съест время)

---

## 2026-05-01: Стек — стейт-менеджер

**Решение:** Zustand
**Почему:** минималистичный (~1KB), нет бойлерплейта, достаточен для хранения активного таймера, текущего вида и фильтров
**Отвергнуто:** Redux Toolkit — избыточен для учебного проекта

## 2026-05-01: Стек — UI-библиотека

**Решение:** Tailwind CSS + shadcn/ui
**Почему:** shadcn копирует компоненты прямо в проект (полный контроль над кодом), Tailwind конфигурируется под нашу дизайн-систему; итог — современный вид без чёрного ящика
**Отвергнуто:** MUI — тяжёлый, навязывает Material Design; чистый CSS — долго с нуля

## 2026-05-01: Стек — дизайн-система

**Решение:** зафиксировали палитру и типографику через ui-ux-pro-max скилл в начале проекта; результат закреплён в `tailwind.config` и `src/index.css`
**Почему:** позволяет один раз зафиксировать палитру, типографику и стиль, не переделывать потом. Скилл удалён после завершения проекта — он одноразовый.
**Отвергнуто:** подбирать цвета вручную по ходу разработки — непредсказуемый результат

## 2026-05-01: План реализации (порядок шагов)

**Решение:** 7 шагов в строгой последовательности:
1. Init Vite + React + TypeScript
2. UI-скилл → дизайн-система (палитра, шрифты, стиль)
3. Tailwind + shadcn/ui с конфигом из дизайн-системы
4. TypeScript-типы (Goal, Project, Task, TimeEntry, Tag, TaskTag)
5. Dexie — схема БД, индексы, проверка сохранения
6. Zustand — стор (таймер, вид, фильтры)
7. Фичи: задачи → таймер → фильтры → экспорт → хоткеи → AI

**Почему такой порядок:** каждый шаг зависит от предыдущего; дизайн до Tailwind, типы до БД, БД до стора, стор до фич

## 2026-05-01: Шаг 2 — Дизайн-система GoalFlow

**Решение:**
- Стиль: Flat Design + Micro-interactions (анимации 50-100ms)
- Шрифт: Inter (Minimal Swiss — для дашбордов и продуктивности)
- Фон: `#FAFAFA` (light) / `#09090B` (dark)
- Поверхности: zinc-900 `#18181B` / zinc-700 `#3F3F46`
- Акцент primary: `#6366F1` (Indigo — схожий с Linear)
- CTA (таймер, ключевые действия): `#F97316` (Orange)
- Текст: `#09090B` — WCAG AAA контраст

**Почему:** zinc-base нейтрален и хорошо ложится на shadcn/ui defaults; indigo читается как «продуктивность»; orange выделяет действия (запуск таймера) без агрессии
**Отвергнуто:** teal-palette — слишком SaaS-корпоративный; чёрный с золотом — слишком editorial, не для таск-менеджера

## 2026-05-01: Роутинг — React Router вместо Zustand-навигации

**Решение:** React Router v6 (`createBrowserRouter`), SPA-rewrite через `vercel.json`
**Почему:** URL-based routing даёт deep linking, корректные Back/Forward, возможность шарить ссылки. `?task=id` query-param открывает TaskDetailPanel без смены маршрута — панель не ломает текущий URL.
**Отвергнуто:** навигация через Zustand-стор — не работает с Vercel (прямой заход на /goals/:id даёт 404), нет Back/Forward браузера

## 2026-05-01: Фазы 1–4 (Роутинг, Sidebar, TaskList, TaskDetailPanel)

**Решение:**
- AppLayout: Sidebar + Header + main + TaskDetailPanel в flex-ряд
- Sidebar: `useLiveQuery` на цели и проекты, GoalItem с click-outside меню, авторасширение по pathname
- TaskItem: клик на заголовок → `setSearchParams({ task: id })`, таймер кнопка visible on hover
- TaskDetailPanel: `w-[400px] shrink-0`, `?task=id` query param, редактирование title/description через onBlur
**Почему:** `setSearchParams((prev) => { prev.set... return prev })` — merge-safe, не сбрасывает другие параметры; `task?.id` в useEffect — сброс локального стейта только при смене задачи, не при каждом DB update

---

## 2026-05-01: Фаза 5 — Таймер в хедере

**Решение:** `setInterval(1000)` в `useEffect([active])` внутри Header, `Math.floor((Date.now() - startedAt) / 1000)`, формат `MM:SS` / `H:MM:SS`, `font-mono tabular-nums` чтобы цифры не прыгали
**Почему:** таймер живёт в компоненте, а не в сторе — elapsed-секунды нужны только UI, не бизнес-логике

---

## 2026-05-01: Фаза 6 — Kanban и фильтры

**Решение:** FilterBar с поиском и фильтром по приоритету (из useUIStore); KanbanBoard с 3 колонками (todo / in_progress / done); статус задачи добавлен в TaskDetailPanel как 3 кнопки
**Почему:** фильтры в Zustand persist (viewMode), но сами значения фильтров не персистируются — при перезагрузке сбрасываются; статус в детальной панели закрывает единственный способ переводить задачи в in_progress

---

## 2026-05-01: Фаза 7 — Drag-and-drop (@dnd-kit)

**Решение:** `@dnd-kit/core` + `@dnd-kit/utilities`; `PointerSensor` с `activationConstraint: { distance: 5 }` чтобы клик на карточку открывал панель, а не запускал drag; `DragOverlay` с `dropAnimation: null` для мгновенного скрытия; статус обновляется через `tasks.update` в `handleDragEnd`
**Отвергнуто:** react-beautiful-dnd (не поддерживается, deprecated), HTML5 DnD API (нет кастомного оверлея)

## 2026-05-01: Деплой — Vercel

**Решение:** Vercel CLI (`vercel --name goalflow`), `vercel.json` с SPA-rewrite `{ "source": "/(.*)", "destination": "/" }`
**Почему:** `vercel.json` rewrite нужен чтобы прямой заход на `/goals/:id` отдавал `index.html`, а не 404 — React Router перехватывает роут на клиенте
**URL:** https://goalflow-sigma.vercel.app

## 2026-05-02: AI-форматирование описания задачи

**Решение:** кнопка «Отформатировать» в `TaskDetailPanel` вызывает LLM, которая структурирует текст описания и возвращает отредактированную версию
**Почему:** пользователи пишут описания в черновом виде; LLM исправляет грамматику, структурирует мысли и делает текст лаконичным — без смены контекста

## 2026-05-02: AI — выбор провайдера (Groq вместо DeepSeek)

**Решение:** Groq API с моделью `llama-3.3-70b-versatile`
**Почему:** DeepSeek API не даёт стартовых бесплатных кредитов по факту — первый запрос вернул `Insufficient Balance`. Groq предоставляет бесплатный tier с высокой скоростью ответа. API совместим с OpenAI SDK — смена провайдера потребовала только замены `baseURL` и `model`.
**Отвергнуто:** DeepSeek (нет бесплатного tier на практике), OpenAI (платный с первого запроса)

---

## 2026-05-02: AI — безопасность API-ключа (Vercel Edge Function)

**Решение:** API-ключ хранится в переменной окружения Vercel (`GROQ_API_KEY`, без `VITE_`-префикса). Браузер вызывает `/api/format-task` — Vercel Edge Function, которая проксирует запрос к Groq. Ключ никогда не попадает в бандл.
**Почему:** `VITE_`-переменные встраиваются в клиентский бандл и видны в DevTools — неприемлемо для production-ключа.
**Отвергнуто:** хранить ключ в `VITE_GROQ_API_KEY` (ключ утекает в браузер)

---

## 2026-05-02: AI — режим локальной разработки

**Решение:** `src/lib/ai.ts` проверяет `import.meta.env.DEV` — в dev-режиме вызывает Groq напрямую из браузера (через `VITE_GROQ_API_KEY` в `.env.local`), в production — через `/api/format-task`.
**Почему:** `vercel dev` не поднимается стабильно в окружении с занятыми портами; `npm run dev` проще и надёжнее для локальной разработки. Прямой вызов из браузера допустим только локально — ключ не попадает в репозиторий (`.env.local` в `.gitignore` через `*.local`).
**Отвергнуто:** `vercel dev` как единственный способ локального запуска — нестабилен при занятых портах

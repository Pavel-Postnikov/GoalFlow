# 00-brief: GoalFlow

## Цель

Создать веб-приложение для управления задачами и временем, которое:
- связывает задачи с целями (понимаешь ЗАЧЕМ делаешь)
- учитывает время без фрикшна (встроено в задачу, не отдельное приложение)
- минималистично (только нужные поля)
- хранит данные локально (твои данные — твои)

## Аудитория

Разработчик-одиночка или небольшая команда, которым нужна чёткая связь между стратегическими целями и ежедневными задачами. Пользователь ценит скорость и клавиатурные шорткаты над кликами мышью.

## Контекст разработки

Студенческий проект Модуля 3 («Персонализация»). Разработчик — Павел, работает в паре с Claude Code. Стек выбирается в процессе проектирования.

## Ключевые требования

1. **Иерархия:** Цели → Проекты → Задачи (возможно Подзадачи)
2. **Учёт времени:** встроенный таймер или ручной ввод прямо в задаче
3. **Виды:** минимум список и канбан; фильтрация по статусу, проекту, цели
4. **Хранение:** локально (localStorage / IndexedDB / SQLite WASM), экспорт JSON/CSV
5. **UX:** хоткеи, quick capture, возможно command palette
6. **AI (опционально):** автокатегоризация, декомпозиция, отчёты

## Ограничения

- Не копировать существующие продукты — понять их философию, сделать своё
- Не overengineering: синхронизация между устройствами не нужна на старте
- Не перегружать UI: каждое поле должно оправдывать своё присутствие
- Деплой необязателен, но будет плюсом

## Вдохновение (философия, не копия)

- **Linear** — скорость, keyboard-first, минимализм
- **Things 3** — чистая иерархия Areas → Projects → Tasks
- **Sunsama** — интеграция времени и задач в одном месте
- **Logseq** — локальные данные, контроль пользователя

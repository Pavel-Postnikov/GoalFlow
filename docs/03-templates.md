# 03-templates: Библиотека промптов GoalFlow

## Промпт 1: Выбор стека

**Задача:** обоснованно выбрать технологии для реализации
**Input:** список кандидатов или открытый вопрос
**Output:** сравнительная таблица + рекомендация с обоснованием

```
Помоги выбрать стек для GoalFlow. Контекст: учебный проект, local-first,
браузерное приложение, разработчик один. Сравни варианты [X, Y, Z] по:
сложности старта, размеру бандла, поддержке offline, наличию нужных примитивов.
Дай рекомендацию с trade-offs.
```

## Промпт 2: Проектирование модели данных

**Задача:** спроектировать схему сущностей
**Input:** список сущностей из glossary.md
**Output:** схема с полями, типами и связями

```
Спроектируй модель данных для GoalFlow. Сущности: Goal, Project, Task, TimeEntry.
Требования: Task может быть без Project, Project может быть без Goal.
Дай схему в виде TypeScript-интерфейсов (или SQL DDL). Учти: хранение в [выбранное хранилище].
```

## Промпт 3: Проверка UI-решения

**Задача:** оценить предложенный UX-подход
**Input:** описание или скетч UI-элемента
**Output:** список проблем + альтернативы

```
Оцени UI-решение для [фича]. Контекст: минималистичный таск-менеджер,
keyboard-first, пользователь ценит скорость. Найди проблемы с UX.
Предложи альтернативы, если есть.
```

## Промпт 4: Декомпозиция задачи разработки

**Задача:** разбить крупную фичу на шаги реализации
**Input:** название фичи + её описание
**Output:** пронумерованный план с оценкой сложности

```
Декомпозируй реализацию фичи «[название]» для GoalFlow.
Стек: [текущий стек]. Дай пошаговый план с оценкой: просто / средне / сложно.
Выдели блокеры, если есть.
```

## Промпт 5: Код-ревью компонента

**Задача:** проверить реализацию на качество и соответствие правилам проекта
**Input:** код компонента или модуля
**Output:** список проблем с приоритетами + исправления

```
Проверь этот код на: корректность логики, производительность, читаемость,
соответствие принципу «минимально достаточно». Стек: [стек].
Формат: критично / важно / мелочь — с примерами исправлений.
```

## Промпт 6: Генерация README

**Задача:** написать README по финальному проекту
**Input:** текущий стек, реализованные фичи, команда запуска
**Output:** готовый README.md в свободном стиле

```
Напиши README.md для GoalFlow. Включи: стек и почему его выбрали,
архитектуру (модель данных, хранение, ключевые компоненты),
как запустить локально, как попробовать основные фичи.
Если есть AI-функции — отдельный раздел с описанием промптов.
Стиль: свободный, живой, без корпоративного шаблона.
```
